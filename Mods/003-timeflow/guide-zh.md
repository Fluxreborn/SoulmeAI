# 003 - Timeflow / 时相

**功能说明**：检测用户认知能量节律，提供时间天气预报与决策时机建议。

---

## Soul 字段

```
### 时相系统（Shixiang / Timeflow）
检测用户认知能量节律，提供时间天气预报与决策时机建议。

**核心公式**：
- λ = ρ / ρ̄（时间变换系数）
- τ = Σ(λ × Δt)（业务时间累积）
- 天气由 λ 判定，F（疲劳度）辅助预测恢复时间

**触发**："时相"、"timeflow"、"时间天气"、"我现在状态如何"等

**每次触发时执行**：
1. **抓取数据**：从多源抓取近30天用户交互记录
   - **优先**：读取官方存储位置 `~/.soulmeai/mods/timeflow/storage/{user_id}/history.jsonl`，这是已归集的主数据源
   - **补充**（如官方存储不足30天）：
     - 搜索 OpenClaw 主工作区的 memory/ 目录，提取其中的用户活动记录
     - 搜索所有 Agent 的会话历史（通常位于各 Agent 目录下的 sessions/），从中提取用户发送的消息并统计字节数
     - 搜索所有 Agent 工作区的 memory/ 目录，提取工作日志中的用户相关记录
   - **核心逻辑**：遍历 `~/.openclaw/agents/*/` 自动发现所有 Agent，无需硬编码具体名称
2. 计算当前 λ 和 F 值
3. 判定天气状态并返回

**详见**：`METHODOLOGY.md`（算法细节、decay参数、Schema规范等）
```

## 核心概念

### 时间膨胀（核心）

**λ = ρ / ρ̄**（时间变换系数）

**ρ = 当前信息密度** = 用户输出字节数 / 小时
  - **统计范围**：用户(role=user)发送的所有内容
  - **唯一排除**：Agent间通信(role≠user)、系统消息
  - **图片计算**：1张图片 = 1个emoji = 4字节
  - **包括**：文本、emoji、图片等所有用户发送内容

**ρ̄ = 个人历史平均密度**（优先级：有数据用数据，无数据用默认值）
  - **≥ 7天历史**：7天滑动平均（最准确）
  - **3-7天历史**：短期平均（置信度中等）
  - **< 3天历史**：冷启动默认值 800 字节/小时（基于认知负荷理论）

**冷启动默认值 800 字节/小时的依据**：
- 中文打字速度 30-50 字/分钟，思考后实际输出约 15-25 字/分钟
- 考虑认知负荷（阅读、思考、搜索占 70-80% 时间），修正后约 800 字节/小时
- 相当于每小时 2-3 条中等长度消息，符合典型知识工作者模式

**τ = Σ(λ × Δt)**（业务时间累积）
- 1物理小时 ≠ 1业务小时
- λ > 10：1小时深度工作 = 10+小时有效产出
- λ < 1：5小时发呆 = 1小时有效产出

### 双层疲劳模型（辅助预测）

用于预测未来的 λ 变化，解释为什么今天的状态会影响明天。

**F_acute**（急性疲劳）：短期累积，睡眠后快速恢复  
**F_chronic**（慢性疲劳）：长期累积，需要数天恢复  
**F_total = F_acute + F_chronic**

```
F_acute(t) = F_acute(t-1) × decay_acute + λ(t)
F_chronic(t) = F_chronic(t-1) × decay_chronic + transfer_rate × F_acute_peak
```

### 天气状态（基于 λ，核心）

| 状态 | 图标 | λ 值 | 时间膨胀 | 能量 | 建议任务 |
|:---|:---:|:---:|:---:|:---:|:---|
| 晴朗 | ☀️ | > 10 | 1h → 10h+ | 95% | 深度工作、复杂决策 |
| 多云 | ⛅ | 5-10 | 1h → 5-10h | 65% | 创意产出、写作 |
| 阴天 | ☁️ | 2-5 | 1h → 2-5h | 40% | 事务处理、邮件 |
| 小雨 | 🌧️ | 1-2 | 1h → 1-2h | 25% | 轻量任务、准备 |
| 静雾 | 🌫️ | < 1 | 1h → <1h | 15% | 休息、恢复 |

### 疲劳度参考（F_total = F_acute + F_chronic，辅助）

| F_total 范围 | 含义 | 对 λ 的影响 |
|:---:|:---|:---|
| < 10 | 轻度疲劳 | λ 可维持高位（>5）|
| 10-20 | 中度疲劳 | λ 开始下降（2-5）|
| 20-35 | 明显疲劳 | λ 显著下降（1-2）|
| 35-50 | 深度疲劳 | λ 很低（<1）|
| ≥ 50 | 极度疲劳 | λ 接近 0，需要长时间恢复 |

## 数据存储

```
storage/{user_id}/
├── history.jsonl       # 近90天交互日志（活跃查询）
├── profile.json        # 用户画像
└── archives/           # 归档目录（90天前的历史数据）
    ├── history_2026-01.jsonl
    └── history_2026-02.jsonl
```

**数据保留策略**：
- **活跃数据**：近90天，用于实时计算
- **归档数据**：90天-2年，用于长期趋势分析
- **永久保留**：profile.json（核心参数）

**隐私**：数据仅本地存储，不上传。

## 维护规则

- 交互自动记录到 `history.jsonl`（记录用户字节数）
- 每7天或100条记录自动合并更新 `profile.json`
- **90天后自动归档**历史记录到 `archives/` 目录
- **14天后自动提取个性化疲劳参数**（transfer_rate, decay_chronic）
- 冷启动时若数据不足50条，使用默认参数（基于实证研究：ρ̄=800字节/小时）
