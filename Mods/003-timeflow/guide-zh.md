# 003 - Timeflow / 时相

**功能说明**：检测用户认知能量节律，提供时间天气预报与决策时机建议。

---

## Soul 字段

```
### 时相系统（Shixiang / Timeflow）
检测用户认知能量节律，提供时间天气预报与决策时机建议。完整方法论见 `Mods/003-timeflow/METHODOLOGY.md`，用户数据存储于 `Mods/003-timeflow/storage/{user_id}/`。

**触发**：用户提及"时相"、"timeflow"、"时间天气"、"能量状态"、"我现在适合做什么"、"该不该现在决策"等意图时，自动读取方法论并执行相应分析。

**特殊集成**：
- 剧场模式前置检查：若当前 F_total > 50（静雾），建议"先休息，明日再决策"
- 若 35 < F_total ≤ 50（小雨），提示"疲劳累积中，简单决策可继续，复杂决策建议延后"
- 若 20 < F_total ≤ 35（阴天），提示"轻度疲劳，30分钟内可决策"
- 若 F_total ≤ 20（晴朗/多云），正常启动剧场对抗
```

## 快速开始

### 1. 首次使用（冷启动）

**用户说**："分析我的时相"

**Agent 执行**：
1. 检查 `storage/{user_id}/profile.json` 是否存在
2. 不存在 → 执行冷启动分析
3. 生成用户画像，返回模式类型（晨型/夜型/下午型）

**Agent 返回**：

## 核心概念

### 信息密度 ρ
单位时间内的有效交互次数（提问、决策、深度回复）

### 时间变换系数 λ
```
λ = ρ / ρ̄

ρ̄ = 个人历史平均密度
```

### 天气状态（v2.0）

| 状态 | 图标 | F_total 范围 | 能量 | 建议任务 |
|:---|:---:|:---:|:---:|:---|
| 晴朗 | ☀️ | < 10 | 95% | 深度工作、复杂决策 |
| 多云 | ⛅ | 10-20 | 65% | 创意产出、写作 |
| 阴天 | ☁️ | 20-35 | 40% | 事务处理、邮件 |
| 小雨 | 🌧️ | 35-50 | 25% | 轻量任务、准备 |
| 静雾 | 🌫️ | > 50 | 15% | 休息、恢复 |

## 数据存储

```
storage/{user_id}/
├── history.jsonl       # 交互日志（append-only）
└── profile.json        # 用户画像
```

**隐私**：数据仅本地存储，不上传。

---

## 维护规则

- 交互自动记录到 `history.jsonl`
- 每7天或100条记录自动合并更新 `profile.json`
- **14天后自动提取个性化疲劳参数**（transfer_rate, decay_chronic）
- 冷启动时若数据不足50条，使用默认参数（基于实证研究）
